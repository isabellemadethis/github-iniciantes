📌 Seção 1

O que é GitHub?

Imagine que você acabou de descobrir um lugar incrível onde as pessoas guardam, organizam e compartilham seus projetos com o mundo. Esse lugar é o GitHub.

🏛️
A Analogia
Uma Biblioteca Pública Gigante
Imagine a maior biblioteca do mundo. Qualquer pessoa pode entrar, pegar um livro para ler e aprender. Autores deixam suas obras lá, e qualquer leitor pode sugerir melhorias nos livros que encontrar.
🐙
Na Prática
GitHub — A Biblioteca de Projetos
O GitHub é um site onde pessoas guardam seus projetos (códigos, trabalhos, estudos). Qualquer um pode ver, copiar e contribuir com projetos de outros. É gratuito e usado por mais de 100 milhões de pessoas no mundo.

📌 Seção 2

O que é um Repositório?

O repositório é a peça central do GitHub. É onde tudo vive: seus arquivos, seu histórico, suas versões. Vamos dissecar um!

😰 Sem GitHub
😱
Você tem 15 arquivos chamados "trabalho_final_v3_revisado_2_ESSE.docx"
🗑️
Você deletou algo importante e não tem como recuperar
😵
Duas pessoas editaram o mesmo arquivo e agora ninguém sabe qual é o certo
💻
O projeto só existe no seu computador. Se ele quebrar, perdeu tudo
📧
Você manda arquivos por e-mail para colaborar com alguém
🚀 Com GitHub
✅
Existe uma única versão atual e todo o histórico de mudanças guardado
⏪
Você pode voltar para qualquer ponto anterior com um clique
🤝
Várias pessoas trabalham juntas sem sobrescrever o trabalho umas das outras
☁️
O projeto fica salvo na nuvem, seguro mesmo se o computador quebrar
🌍
Qualquer pessoa no mundo pode ver e contribuir com seu projeto (se você quiser)

📌 Seção 3

Os 6 Conceitos Essenciais

Cada termo do GitHub tem uma analogia perfeita do dia a dia. Uma vez que você entender esses 6, você está pronto para começar!

📸
Conceito 1
Commit
É como...
"Tirar uma foto do seu trabalho"
Quando você salva uma versão do seu projeto com uma mensagem explicando o que mudou. Ex: "Adicionei o título da página". Cada commit fica guardado para sempre no histórico — você pode voltar para qualquer foto antiga.
🌿
Conceito 2
Branch
É como...
"Um caderno de rascunho separado"
Você cria um "rascunho" do projeto para testar algo novo sem estragar a versão principal. Se ficar bom, você mistura de volta (merge). Se não ficou legal, você descarta sem prejudicar nada.
🔀
Conceito 3
Merge
É como...
"Misturar o rascunho ao original"
Depois de terminar as mudanças no seu rascunho (branch), você "mistura" tudo de volta para a versão principal. O Git é inteligente e faz isso automaticamente na maioria dos casos.
🍴
Conceito 4
Fork
É como...
"Xerocopiar o livro de outra pessoa"
Você pega o projeto de outra pessoa, faz uma cópia completa para a sua conta, e pode editar à vontade. O projeto original não é afetado. Muito usado para aprender com projetos reais ou contribuir com projetos open-source.
🙋
Conceito 5
Pull Request
É como...
"Levantar a mão e dizer: 'quero contribuir!'"
Quando você fez mudanças num projeto (seu ou alheio) e quer que essas mudanças sejam incluídas na versão principal. É como enviar uma sugestão formal — o dono do projeto revisa, aprova ou pede ajustes antes de aceitar.
⬇️
Conceito 6
Clone
É como...
"Baixar o livro para ler em casa"
Você baixa uma cópia completa do repositório para o seu computador. Diferente do Fork (que fica no GitHub), o Clone fica na sua máquina. Você trabalha localmente e depois envia as mudanças de volta ao GitHub.
📌 Seção 4

Como Funciona na Prática?

O fluxo básico do GitHub é um ciclo simples de 4 etapas que você vai repetir toda vez que trabalhar num projeto.

O Ciclo Básico do Git + GitHub
No seu Computador
💻
Trabalhar
Você cria ou edita arquivos no seu computador normalmente
→
git add
Preparar
Área de Preparo
📦
Separar
Você escolhe quais mudanças quer salvar agora
→
git commit
Fotografar
Repositório Local
📸
Salvar
A versão é salva com uma mensagem no histórico
→
git push
Enviar
GitHub.com
🐙
Publicar
Suas mudanças chegam ao GitHub e ficam salvas na nuvem
git pull
← Quando você quer baixar as mudanças que outra pessoa fez no GitHub para o seu computador
🛠️
Ferramenta Local
Git
É o programa instalado no seu computador. Ele controla versões, cria commits e gerencia branches. Funciona sem internet. É como o mecanismo de uma câmera fotográfica.
🌐
Plataforma Online
GitHub
É o site (github.com) onde você publica seus repositórios. Funciona como a nuvem onde suas "fotos" ficam salvas, e onde outras pessoas podem ver e colaborar. É como o álbum de fotos online.
📌 Seção 5

Seus Primeiros Passos 🚀

Siga esse roteiro para criar sua conta, seu primeiro repositório e fazer seu primeiro commit!

1
🌐 Crie sua conta no GitHub
Acesse github.com e clique em "Sign up". Escolha um nome de usuário que você usará profissionalmente (ele aparece no seu portfólio!). O plano gratuito é suficiente para tudo que você vai fazer.
💡 Dica: Use um e-mail profissional que você sempre terá acesso
2
💻 Instale o Git no seu computador
O GitHub é o site, mas o Git é o programa que você instala. Baixe em git-scm.com. Depois de instalar, configure com seus dados:
$ git config --global user.name "Seu Nome" ← seu nome
$ git config --global user.email "seu@email.com" ← seu e-mail
3
📁 Crie seu Primeiro Repositório
No GitHub, clique no botão verde "New" (ou "+" no canto superior). Dê um nome ao repositório, marque "Add a README file" e clique em "Create repository".
💡 Público = qualquer pessoa vê · Privado = só você (e quem você convidar)
4
⬇️ Clone o repositório para o seu computador
Copie o link do seu repositório e execute no terminal. Isso baixa a pasta do projeto para o seu computador:
$ git clone https://github.com/seu-usuario/seu-projeto.git
$ cd seu-projeto ← entra na pasta
5
✏️ Faça mudanças e seu Primeiro Commit
Edite algum arquivo (pode ser o README.md), depois prepare e salve a versão:
$ git add . ← prepara TODAS as mudanças
$ git commit -m "Meu primeiro commit!" ← salva com mensagem
$ git push ← envia para o GitHub 🎉
🎉
Parabéns!
Você acabou de fazer seu primeiro commit no GitHub. Agora você faz parte de uma comunidade de mais de 100 milhões de pessoas!
Vá em github.com/seu-usuario para ver seu repositório publicado para o mundo.
📌 Seção 6

Glossário Rápido

Uma colinha dos termos mais importantes. Guarde essa tabela para consultar sempre que precisar!

Termo Analogia do Dia a Dia O que é de Verdade
repository 📚 Um livro / pasta de projeto Pasta que contém todos os arquivos do projeto + histórico completo de mudanças
commit 📸 Tirar uma foto do projeto Salvar uma "versão" com uma mensagem descrevendo o que mudou
branch 📓 Caderno de rascunho Uma cópia paralela do projeto para testar mudanças sem afetar a versão principal
main / master 📖 O livro original A branch principal — a versão "oficial" e estável do projeto
merge 🔀 Misturar rascunho ao original Juntar as mudanças de uma branch de volta à branch principal
fork 📋 Xerocopiar o livro alheio Copiar o repositório de outra pessoa para a sua conta do GitHub
clone ⬇️ Baixar o livro da biblioteca Copiar um repositório do GitHub para o seu computador
push 📤 Devolver o livro com suas anotações Enviar seus commits locais para o GitHub
pull 📥 Pegar o livro atualizado Baixar as mudanças do GitHub para o seu computador
pull request 🙋 Levantar a mão para sugerir Pedido formal para incluir suas mudanças no projeto de alguém
README 📄 A capa e o índice do livro Arquivo de apresentação do projeto — primeira coisa que as pessoas veem
open source 🔓 Livro com licença aberta Projeto com código público que qualquer pessoa pode usar, estudar e contribuir
issue 🐛 Post-it de problema/sugestão Um relatório de bug ou sugestão de melhoria num projeto
star ⭐ ❤️ Favoritar um livro Marcar um repositório como favorito (também indica popularidade)
📌 Seção 7

Dicas para o Seu Dia a Dia

Hábitos simples que vão fazer uma diferença enorme nos seus estudos e projetos.

📝
Escreva boas mensagens de commit
Em vez de "atualizações", escreva "Adiciona formulário de contato na página principal". Mensagens claras facilitam muito quando você precisar encontrar uma mudança específica no futuro.
🔖
Faça commits pequenos e frequentes
Não espere terminar tudo para commitar. Salve versões a cada pequena conquista, como capítulos de um livro. Isso facilita identificar onde algo deu errado.
📖
Sempre escreva um bom README
O README.md é a primeira coisa que as pessoas veem. Explique o que é o projeto, como usar e para que serve. É o seu cartão de visitas para recrutadores e colaboradores.
🌿
Use branches para tudo novo
Quer testar uma nova funcionalidade? Crie uma branch! Assim a versão principal fica sempre intocada e funcional enquanto você experimenta.
⭐
Explore repositórios famosos
Navegue pelo GitHub, dê estrelas nos projetos que achar interessante e estude como pessoas experientes organizam seus projetos. É a melhor escola gratuita do mundo.
🟩
Contribua todo dia (o gráfico verde)
Seu perfil mostra um gráfico de contribuições diárias. Commitar todos os dias, mesmo pequenos avanços, mostra consistência — e fica ótimo para empregadores verem!
🤝
Contribua com projetos open-source
Comece corrigindo erros de digitação em documentações, ou traduzindo textos. É uma forma incrível de aprender na prática e aparecer no radar de projetos reais.
🔍
Use git status antes de tudo
Antes de commitar, sempre rode git status para ver exatamente o que você está prestes a salvar. Evita muitos erros!
